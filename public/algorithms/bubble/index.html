Bubble Sort

The bubble sort is the most intuitive sorting algorithm there is.

It works by:

  • iterating over the entire array.
  • repeatedly swapping adjacent elements if they are out of order.
  • with each iteration, the end index is decremented.

After each iteration, one more item at the end of the array is in its final place.

There is an easy way to remember what the bubble sort is. I like to think of the right side of the array as a growing “bubble” of sorted elements (maximums if you are sorting in ascending order).

The bubble sort is:

  • “stable”, if two elements are equal, their relative order will not change.
  • “in-place”, meaning it does not require extra memory to run.
  • not efficient, having O(n^2) performance.
  • not possible to parallelize.
  • works with non-numerical types, it uses a comparator.

It is possible to adapt the bubble sort so that between each iteration, it checks if the array is already sorted, and it could stop early.
